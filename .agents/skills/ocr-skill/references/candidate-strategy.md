# 候选人匹配与投递策略

用于岗位匹配、公司排序、投递优先级和短期行动计划。

## 信息收集

先确认：

- 年级与毕业时间；
- 学历、学校、专业；
- 技术栈和熟练度；
- 项目、实习、竞赛、科研、开源经历；
- 目标城市、可实习时长、到岗时间；
- 目标岗位方向和不想投的方向；
- 当前简历版本和准备时间。

信息不足时可以先给暂定判断，但要标注“需要补充的信息”。

## 匹配维度

使用七个维度，0-3 分只作为内部判断辅助：

```text
基础资格匹配
岗位技能匹配
项目相关度
实习/工程经历
竞争力加分项
城市与投递约束
准备成本与时间窗口
```

分数含义：

```text
0 = 明显不匹配或信息缺失严重
1 = 弱匹配，需要大量补强
2 = 基本匹配，可以投递但需要优化表达
3 = 强匹配，应优先投递
```

不要用总分替代解释，必须说明证据。

## 岗位技能信号

- 前端：HTML / CSS / JavaScript / TypeScript、React / Vue、小程序、工程化、性能优化、组件抽象。
- 后端：Java / Go / C++ / Python、数据库、缓存、消息队列、分布式、服务治理。
- 客户端：Android / iOS / Flutter、网络、端侧性能、稳定性。
- AI 应用 / 大模型应用：Python、LLM API、RAG、Agent、向量数据库、Prompt、后端或前端集成、评估、成本。
- 算法 / 机器学习：数学基础、机器学习 / 深度学习、论文、竞赛、科研或算法项目。
- 数据开发：SQL、数仓、Spark / Flink、ETL、指标体系。
- 测试开发：自动化测试、接口测试、质量平台、脚本开发。
- SRE / 运维开发：Linux、网络、容器、Kubernetes、监控、自动化。
- 安全：Web 安全、二进制、安全工具、漏洞挖掘、攻防经验。

## 推荐层级

使用：

```text
主投：基础匹配，准备成本可控，简历能快速对齐。
冲刺：岗位有吸引力但存在明显短板，需要集中补强。
低优先级：硬性条件、技能栈或时间窗口不适合。
```

不要使用“保底”。需要表达相对可行时，用“相对稳妥”。

## 投递排序

排序依据：

1. 硬性条件匹配；
2. 简历与岗位的证据密度；
3. 截止时间紧急程度；
4. 准备成本；
5. 公司、城市、业务偏好；
6. 竞争强度。

简历版本建议控制在 2-3 个：

- 主方向版本；
- 相邻方向版本；
- 特定公司或 JD 定制版本。

## 输出规则

建议用以下结构：

```text
结论：
主投方向：
冲刺方向：
低优先级方向：
依据：
主要短板：
补强建议：
投递策略：
接下来 7 天行动：
```

如果使用 `scripts/score_match.py`，输出时说明它只是轻量辅助评分，最终结论以证据解释为准。
