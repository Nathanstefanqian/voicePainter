# JD 分析模块

用于拆解用户提供的岗位描述、判断岗位族、提取准备重点。

## 分析顺序

1. 判断岗位族。
2. 拆分岗位职责和任职要求。
3. 提取硬性要求。
4. 提取加分项。
5. 提取技术关键词。
6. 推断隐含能力，并明确标注为推断。
7. 判断简历应突出什么。
8. 预测面试追问方向。

## 硬性要求识别

常见硬性要求包括：

- 毕业年份、学历、专业；
- 必须掌握的语言、框架、平台；
- 实习时长、到岗时间、工作城市；
- 项目、竞赛、论文、开源或业务经验；
- 岗位特殊限制，如必须线下、必须转正实习。

## 加分项识别

常见加分项包括：

- 大厂、AI 公司或强相关业务实习；
- 开源项目、竞赛、论文、专利；
- 真实线上项目、用户规模、性能指标、成本指标；
- 跨端、工程化、复杂业务、基础设施经验；
- RAG、Agent、工具调用、评估体系、LLMOps 等 AI 应用落地经验。

## 隐含能力推断

以下内容只能作为推断，不能伪装成 JD 明示要求：

- `复杂业务系统`：抽象能力、工程能力、协作能力。
- `高并发/海量数据`：性能、稳定性、分布式系统理解。
- `AI 应用落地`：工程集成、评估、成本控制、异常处理。
- `增长/推荐/广告`：数据意识、实验意识、业务指标理解。
- `基础架构`：系统设计、可靠性、工程基本功。

## 输出模板

```text
岗位定位：
核心职责：
硬性要求：
加分项：
技术关键词：
根据 JD 推断：
简历应突出：
面试可能追问：
准备优先级：
```

回答中要把“JD 明示内容”和“基于经验的推断”分开。
